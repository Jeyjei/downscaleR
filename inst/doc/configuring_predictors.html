Preparing predictors in perfect prog downscaling experiments

J. Bedia (Santander Meteorology Group)

2017-08-24

Introduction

This is a brief document illustrating the preparation of different predictor configurations in perfect-prog experiments.

Broadly speaking, there are two main configurations: 1. Using atmospheric fields “as they are” for a given spatial domain 2. Using principal components obtained from these fields. This can be either a principal component calculated upon a particular variable, or a combined PC considering a combination of different predictor variables.

Furthermore, in addition to Principal Components that provide a synoptic descriptor, local information can be also as predictor, providing added information to the downscaling method for the surroundings of the target location for a particualr variable (e.g. the surface temperature in the nearest grid cells around a given station).

The potentially large number of options for a fine-tuning of a downscaling method requires a flexible, yet easily configurable interface enabling users to launch complex experiments testing different predictor setups. prepare_predictors has been designed to this aim. A few reproducible examples are presented in this vignette.

A note of the terminology used

In the climate4R bundle (see e.g. Cofiño et al. 2017), atmospheric variables are stored in the so called data grids. In order to efficiently handle multiple variables used as predictors in downscaling experiments, “stacks” of grids are used. These are known as multiGrids, and can be obtained using the constructor makeMultiGrid from a set of -dimensionally consistent- grids.

Data

Predictors

Daily data from the NCEP reanalysis (Kalnay et al. 1996) are used as example. In particular, a domain centered on the Iberian Peninsula is considered, and three variables (mean sea-level pressure psl, specific humidity at 850mb hus850 and air temperature at 850mb ta850) will be used as predictors. The grids are already spatially and temporally consistent, so they can be stacked in a multiGrid structure:

x <- makeMultiGrid(NCEP_Iberia_hus850, NCEP_Iberia_psl, NCEP_Iberia_ta850)

Predictands

The predictands correspond to the observations used for downscaling. These are typically meteorological station data or interpolated gridded datasets. In this example, we use a subset of stations used in a large intercomparison experiment of downscaling methods performed in the framework of the COST Action VALUE (Maraun et al. 2015). The target variable is daily precipitation.

data("VALUE_Iberia_tp")
y <- VALUE_Iberia_tp
plotClimatology(climatology(y), backdrop.theme = "countries", cex = 1.5,
                main = "Mean Winter daily precip (mm/day, 1983-2002")
## [2017-08-24 13:29:23] - Computing climatology...
## [2017-08-24 13:29:23] - Done.

Precipitation observations used as predictand. Type help("VALUE_Iberia_tp") for further details.

Although the aim of this example is preparing only the predictors (the predictands are not manipulated), the information of the predictand is always required in order to ensure the spatio-temporal consistency of the experiment. The function handles internally non-overlapping temporal periods, ensuring that there is a perfect match between predictors and predictand prior to model calibration.

Using the raw predictor variables

In this situation, the multigrid is directly passed to prepare_predictors. Note that only x (predictors multigrid) and y (predictand, stations) are passed to the function, while the rest of arguments are left with their default value NULL (they could be omitted):

out <- prepare_predictors(x = x,
                          y = y,
                          subset.vars = NULL,
                          PCA = NULL,
                          local.predictors = NULL)

The output object is a list with different components:

  • global: Is the global predictor data matrix. In this example, it is a matrix in which rows correspond to times (days in this case) and the columns to locations, arranged via the internal helper transformeR::array3Dto2Dmat. If there is more than one variable as predictor, as in the example, the matrices for each variable are just binded by columns. In other cases, fopr instance when using PCs, this matrix corresponds to the selected combined PC (rows are times, and columns are PCs, in decreasing order of explained variance according to the user specifications).
  • local: in case local predictors are used, the local information is included here. This is a list of matrices, of the same longitude as the number of locations in the predictand, each one containing the local information that will be appended to the global matrix.
  • pca: This element stores the whole PCA (this is the output of the transformeR::prinComp function). It contains metadata required in subsequent steps.

In addition, several attributes are appended to the output to aid in the traceability of the predictor structure:

str(attributes(out))

Using PCs as predictors

In this example, instead of using the raw fields as predictors, we will use principal components. The tuning of the principal component analysis can be undertaken by passing the different possible arguments admitted by transformeR::prinComp, which are detailed in the help of the function to the argument PCA of prepare_predictors, in the form of a named list. Note that here we will use the first 5 PCs of the input variables contained in the multigrid. This applies also to the combined PC, that is used as predictor

out <- prepare_predictors(x = x,
                          y = y,
                          PCA = list(n.eofs = 5, combined.PC = TRUE)
)
## [2017-08-24 13:29:27] Performing PC analysis on 3 variables plus a combination ...
## [2017-08-24 13:29:27] Done.

In this case, the output element pca contains the full output of prinComp, that will be needed in subsequent steps of the downscaling. There is also a global attribute PCA.pars containing metadata of the PCA options chosen. Note that now, the global element of the output contains the combined PC matrix (in this example it contains 5 columns, that is, the number of PCs retained as indicated in the argument n.eofs).

Introducing local predictors

Finally, it is possible to specify local predictors using the corresponding argument. It is passed as a named list with the following elements:

  • neigh.vars: the names of the variables to be used as local predictors
  • n.neighs: the number of nearest points/grid-boxes to the predcitand location to be used
  • neigh.fun: the aggregation function of the selected neighbours (if any). If NULL (the default), the nearest neighbours are not aggregated but just appended to the predictor matrix.

TIP: In order to select variables, either for local predictors or for multigrid subsetting via subset.vars, the helper getVarNames from transformeR may be useful:

getVarNames(x)
## [1] "hus850" "psl"    "ta850"

Next, we will use the raw fields of air temperature at 850mb and sea-level pressure as global predictors (subset.vars = c("ta850", "psl")). In addition, specific humidity at 850mb will be used as a local predictor (neigh.vars = "hus850"). In this example, we will consider the 5 closest points to the predictand location (n.neighs = 5), and no aggregation of the neighbours is undertaken (neigh.fun = NULL, this argument could be ommited as this is the default).

out <- prepare_predictors(x = x,
                          y = y,
                          subset.vars = c("ta850", "psl"),
                          local.predictors = list(neigh.vars = "hus850",
                                                  n.neighs = 5,
                                                  neigh.fun = NULL)
)

Instead of using all 5 neighbours sepparately, these can be aggregated, for instance using their average value as local predictor (neigh.fun = list(FUN = "mean")), or any other user-defined function, by adding all the additional arguments needed in the neigh.fun list (for instance, the 90th percentile value would be specified as neigh.fun = list(FUN = "quantile", prob = .9), etc).

out <- prepare_predictors(x = x,
                          y = y,
                          subset.vars = c("ta850", "psl"),
                          local.predictors = list(neigh.vars = "hus850",
                                                  n.neighs = 5,
                                                  neigh.fun = list(FUN = "mean"))
)

Note that the same predictor configuration could be used but calculating the combined PC of the global predictors intead of their raw values. In this case, we indicate that we want to keep the PCs explaining no less than 97.5% of the total variance (v.exp = 0.975; note that in the previous PCA example we indicated the number of PCs, instead of the amount of explained variance):

out <- prepare_predictors(x = x,
                          y = y,
                          subset.vars = c("ta850", "psl"),
                          PCA = list(v.exp = .975, combined.PC = TRUE),
                          local.predictors = list(neigh.vars = "hus850",
                                                  n.neighs = 5,
                                                  neigh.fun = NULL)
)
